History of CSS

  • Style sheets have existed in one form or another since the 80s.
  • CSS language first proposed in 1994.
  • First W3C CSS Recommendation released in 1996.
  • Fully adopted by a browser in 2000: CSS 1 - IE 5 for Mac (!)
  • CSS 2.1 published as a W3C Recommendation in 2011.
  • CSS3 Modules are published as a W3C Reccomendation since 2011.

  • One requirement for a web style sheet language was for style sheets to come from different sources.

    most of the existing style sheet languages were unable to meet that requirement.

    CSS, on the other hand, allowed a document's style to be influenced by multiple style sheets, this is referred as cascading.

  • Discussions on public mailing lists and inside W3C resulted in the first W3C CSS Recommendation being released in 1996.

  • CSS 3 has a modular approach.

    CSS3, modules: each of these modules is now an independent part of the language and moves towards standardization at its own pace. While some modules are already W3C Recommendations, other still are early Working Drafts.

    Formally, there is no CSS3 standard per se. Each module being standardized independently, the standard CSS consists of CSS2.1 amended and extended by the completed modules, not necessary all with the same level number.

Cascading Style Sheets (CSS)

a stylesheet language designed primarily to enable the separation of document content from document presentation

in a perfect world

In front-end web development, separating content from presentation allows you the freedom to redesign an entire site without touching the HTML.

CSS ZenGarden expressed this beautifully in 2003

In the real world

Presentation (CSS) is pointless without structure (HTML) and it's pointless to separate the content from structure.

The HTML is built around the content.
Content contain structure (<ul></ul>) and structure contain presentation (presentational ids)
Presentation can and should be separated from content.
