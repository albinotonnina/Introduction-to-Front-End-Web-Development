The section element


        <body>
            ...
            <section>
                
level of heading = section nesting level

                rest of the content
            </section>
            ...
        </body>
  • is not a generic container element. The <div> is.
  • is a blob of content that you could store as an individual record in a database.
  • with very few exceptions, section should not be used if there is no natural heading for it.
  • don’t use it just as hook for styling or scripting; that’s a div.
  • don’t use it if article, aside or nav is more appropriate.

The figure & figcaption elements


    

    <figure>
      <img src="/kookaburra.jpg" alt="Kooaburra">
      <img src="/pelican.jpg" alt="Pelican stood on the beach">
      <img src="/lorikeet.jpg" alt="Cheeky looking Rainbow Lorikeet">
      <figcaption>Australian Birds. From left to right, Kookburra, Pelican and Rainbow Lorikeet. Originals by Richard Clark</figcaption>
    </figure>
  • represents a unit of content that can be moved away from the main flow of the document without affecting the document’s meaning.
  • introduced because in traditional printed material like books and magazines, an image, chart, or code example would be accompanied by a caption.
